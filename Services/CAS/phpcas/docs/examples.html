phpCAS examples

A simple CAS client

phpCAS can be used the simplest way, as a CAS client.

  • A simple CAS client: examples/example_simple.php

Run-time behaviour configuration

When setting up a CAS proxy client, some runtime behaviour can be easily configured.

  • Language: examples/example_lang.php
  • HTML output: examples/example_html.php

A CAS proxy

phpCAS can also make a PHP script act as a CAS proxy (calling external services).

  • A CAS proxy : examples/example_proxy.php
  • A CAS proxied client: examples/example_service.php
  • CAS proxies can be chained (a CAS proxied client can also be a proxy itself): examples/example_proxy2.php

PGT storage configuration

PGT storage can be easily configured.

  • Onto the filesystem: examples/example_file.php
  • Into a database: examples/example_db.php

Sessioning between CAS proxies and services

Cookies sent by services are memorized by CAS proxies, which permits sessioning. An example of this feature can be for instance counting the number of requests performed by the CAS proxy to the client:

  • examples/example_session_service.php

At the proxy level, nothing more has to be done:

  • examples/example_session_proxy.php

Only check authentication (gateway)

The possibility of using the CAS gateway feature (see http://tp.its.yale.edu/tiki/tiki-index.php?page=CAS+Gateway) was added in release 0.4.20.

  • examples/example_gateway.php
